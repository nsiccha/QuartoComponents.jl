QuartoComponents.jl

Author

Nikolas Siccha

Allows programmatically generating some Quarto components, which may include GR or PlotlyJS plots. See https://nsiccha.github.io/QuartoComponents.jl/#example for an example.

Current features

Allows defining the following elements:

  • Container(content) (content should be iterable)
  • Heading(level, content)
  • Div(header, content)
  • Code(header, content)
  • Tabset(content) (content should support pairs(content))
  • Lazy rendering of HTML plots (see https://nsiccha.github.io/QuartoComponents.jl/#lazy-rendering). This hopefully makes quarto rendering of many Plotly plots faster.

Currently nothing gets exported.

Current limitations

I’m sure there’s a way around that, but for PlotlyJS plots to render, you have to potentially manually ensure that require.js is loaded. This can be done by adding the following to the Quarto YAML header, though I’m sure there’s a better way to do it:

format: 
    html:
        include-in-header:
            text: |
                <script src="https://cdnjs.cloudflare.com/ajax/libs/require.js/2.3.6/require.min.js" integrity="sha512-c3Nl8+7g4LMSTdrm621y7kf9v3SDPnhxLNhcjFJbKECVnmZHTdo+IRO05sNLTH/D3vA6u1X32ehoLC7WFVdheg==" crossorigin="anonymous"></script>

## Example

```{julia}
#| echo: false

using QuartoComponents, Plots, Base64
plotlyjs();
 

p1 = scatter(randn(10), randn(10))
 
p2 = scatter(randn(10), randn(10))

nothing
```

### Direct rendering

```{julia}
QuartoComponents.islazy!(false)
QuartoComponents.Tabset((;p1,p2))
```

### Lazy rendering

Lazyness can be set internally via `QuartoComponents.islazy!` externally via the environment variable `QUARTOCOMPONENTS_LAZY` 

```{julia}
QuartoComponents.islazy!(true)
QuartoComponents.Tabset((;p1,p2)) 
```
